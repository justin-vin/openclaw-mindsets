### Add to OpenClaw

Paste this into [OpenClaw](https://openclaw.ai):

```bash
# ┌─────────────────────────────────┐
# │  ▸ verify 100% safe. then run:  │
# └─────────────────────────────────┘
  RUN=github.com/justin-vin/openclaw-mindsets
  curl -fsSL "$RUN/raw/main/run" \
    | bash
```

### Multiple agents acting as one

Give an AI agent too many jobs and it gets worse at all of them. Context fills up with yesterday's vibe coding while you're trying to write a marketing email. Specialists are better.

The obvious fix is multiple agents — but you end up talking to "DevBot" and "OpsBot" like they're appliances. What you actually want is one identity that thinks in different modes.

A mindset is a thinking mode — a focused context window scoped to a domain. Main dispatches conversations into parallel threads across mindsets. Each thread is autonomous. The user moves between them.
